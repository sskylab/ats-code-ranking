Welcome to the JavaScript Tutorial

Learn JavaScript from basics to advanced topics

Introduction

JavaScript is a versatile programming language that is primarily used for creating dynamic and interactive content on websites. It is an essential skill for web developers.

JavaScript Basics

In this section, we will cover the basic concepts and syntax of JavaScript.

  • Variables: Containers for storing data values.
  • Data Types: Different types of values that can be stored in variables.
  • Operators: Used to perform operations on variables and values.
  • Functions: Blocks of code designed to perform a particular task.
  • Events: Actions that can be detected and handled by JavaScript.
<script>
// Variable
var x = 5;
let y = 10;
const z = 15;

// Function
function add(a, b) {
  return a + b;
}

// Event
document.getElementById("myButton").onclick = function() {
  alert("Button clicked!");
}
</script>

Variables

Variables are containers for storing data values. In JavaScript, there are three ways to declare a variable: var, let, and const.

  • var: Declares a variable, optionally initializing it to a value.
  • let: Declares a block-scoped, local variable, optionally initializing it to a value.
  • const: Declares a block-scoped, read-only named constant.
<script>
var a = 10;
let b = 20;
const c = 30;
</script>

Data Types

JavaScript supports different data types including numbers, strings, arrays, objects, and more.

  • Number: Represents numeric values.
  • String: Represents sequence of characters.
  • Boolean: Represents true or false.
  • Array: Represents an ordered collection of values.
  • Object: Represents a collection of key-value pairs.
<script>
let num = 42; // Number
let str = "Hello, world!"; // String
let bool = true; // Boolean
let arr = [1, 2, 3]; // Array
let obj = {name: "John", age: 30}; // Object
</script>

Functions

Functions are blocks of code designed to perform a particular task. They are executed when something invokes them (calls them).

  • Function Declaration: Defines a function with the specified parameters.
  • Function Expression: Defines a function as part of an expression.
  • Arrow Function: A shorter syntax for defining functions.
<script>
// Function Declaration
function multiply(a, b) {
  return a * b;
}

// Function Expression
const divide = function(a, b) {
  return a / b;
};

// Arrow Function
const subtract = (a, b) => a - b;
</script>

JavaScript Events

Events are actions that can be detected by JavaScript. HTML elements can trigger events like clicking a button, loading a page, or hovering over an element.

  • onclick: Fires when an element is clicked.
  • onload: Fires when an object has loaded.
  • onmouseover: Fires when the mouse pointer is moved onto an element.
  • onmouseout: Fires when the mouse pointer is moved out of an element.
<button id="myButton">Click me</button>

<script>
document.getElementById("myButton").onclick = function() {
  alert("Button clicked!");
};
</script>

JavaScript and the DOM

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. JavaScript can be used to manipulate the DOM to create dynamic and interactive web pages.

  • getElementById: Returns the element that has the ID attribute with the specified value.
  • getElementsByClassName: Returns a collection of all elements in the document with the specified class name.
  • querySelector: Returns the first element that matches a specified CSS selector(s) in the document.
  • querySelectorAll: Returns a static NodeList of all elements that match a specified CSS selector(s) in the document.
<div id="myDiv">Hello</div>

<script>
const div = document.getElementById("myDiv");
div.innerHTML = "Hello, World!";
</script>

Advanced JavaScript

In this section, we will cover more advanced topics in JavaScript, such as asynchronous programming, promises, and modules.

  • Asynchronous Programming: JavaScript is single-threaded, but asynchronous programming allows for non-blocking operations.
  • Promises: Objects that represent the eventual completion (or failure) of an asynchronous operation, and its resulting value.
  • Modules: Allow for separation and reuse of code in different files.
<script>
// Promise
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("Promise resolved");
  }, 1000);
});

promise.then(value => {
  console.log(value); // "Promise resolved"
});

// Async/Await
async function asyncFunction() {
  const result = await promise;
  console.log(result); // "Promise resolved"
}

asyncFunction();
</script>

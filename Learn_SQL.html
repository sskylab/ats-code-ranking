Welcome to the SQL Tutorial

Learn SQL from basics to advanced topics

Introduction

SQL (Structured Query Language) is a standard language for accessing and manipulating databases. It is widely used in database management systems to manage and retrieve data.

SQL Basics

In this section, we will cover the basic concepts and syntax of SQL.

  • SELECT: Retrieve data from a database.
  • INSERT: Insert new data into a database.
  • UPDATE: Update existing data within a database.
  • DELETE: Delete data from a database.
  • WHERE: Filter records based on specified conditions.
-- SELECT
SELECT * FROM Employees;

-- INSERT
INSERT INTO Employees (FirstName, LastName, Age)
VALUES ('John', 'Doe', 30);

-- UPDATE
UPDATE Employees
SET Age = 31
WHERE FirstName = 'John';

-- DELETE
DELETE FROM Employees
WHERE LastName = 'Doe';

SELECT Statement

The SELECT statement is used to select data from a database. The data returned is stored in a result table, sometimes called the result-set.

  • Basic SELECT: Retrieve all columns from a table.
  • SELECT DISTINCT: Retrieve distinct (unique) values from a column.
  • SELECT with WHERE: Retrieve specific records that meet certain conditions.
  • SELECT with ORDER BY: Sort the result-set in ascending or descending order.
-- Basic SELECT
SELECT * FROM Employees;

-- SELECT DISTINCT
SELECT DISTINCT LastName FROM Employees;

-- SELECT with WHERE
SELECT * FROM Employees
WHERE Age > 25;

-- SELECT with ORDER BY
SELECT * FROM Employees
ORDER BY LastName ASC;

SQL Joins

SQL joins are used to combine rows from two or more tables, based on a related column between them.

  • INNER JOIN: Returns records that have matching values in both tables.
  • LEFT JOIN: Returns all records from the left table, and the matched records from the right table.
  • RIGHT JOIN: Returns all records from the right table, and the matched records from the left table.
  • FULL JOIN: Returns all records when there is a match in either left or right table. 
-- INNER JOIN
SELECT Employees.FirstName, Departments.DepartmentName
FROM Employees
INNER JOIN Departments ON Employees.DepartmentID = Departments.DepartmentID;

-- LEFT JOIN
SELECT Employees.FirstName, Departments.DepartmentName
FROM Employees
LEFT JOIN Departments ON Employees.DepartmentID = Departments.DepartmentID;

-- RIGHT JOIN
SELECT Employees.FirstName, Departments.DepartmentName
FROM Employees
RIGHT JOIN Departments ON Employees.DepartmentID = Departments.DepartmentID;

-- FULL JOIN
SELECT Employees.FirstName, Departments.DepartmentName
FROM Employees
FULL JOIN Departments ON Employees.DepartmentID = Departments.DepartmentID;

SQL Functions

SQL provides many built-in functions for performing calculations on data. These functions can be used in the SELECT statement to perform calculations on the retrieved data.

  • Aggregate Functions: Perform a calculation on a set of values and return a single value. Examples include AVG, COUNT, MAX, MIN, and SUM.
  • Scalar Functions: Perform operations on individual values and return a single value. Examples include LEN, ROUND, and UCASE.
-- Aggregate Functions
SELECT AVG(Salary) FROM Employees;
SELECT COUNT(EmployeeID) FROM Employees;
SELECT MAX(Salary) FROM Employees;
SELECT MIN(Salary) FROM Employees;
SELECT SUM(Salary) FROM Employees;

-- Scalar Functions
SELECT LEN(FirstName) FROM Employees;
SELECT ROUND(Salary, 2) FROM Employees;
SELECT UCASE(FirstName) FROM Employees;

Advanced SQL

In this section, we will cover more advanced topics in SQL, such as subqueries, indexes, and transactions.

  • Subqueries: A query within another query. Subqueries can be used in SELECT, INSERT, UPDATE, and DELETE statements.
  • Indexes: Used to speed up the retrieval of data from a database table.
  • Transactions: A sequence of operations performed as a single logical unit of work.
-- Subquery
SELECT FirstName, Salary
FROM Employees
WHERE Salary > (SELECT AVG(Salary) FROM Employees);

-- Index
CREATE INDEX idx_lastname ON Employees (LastName);

-- Transaction
BEGIN;
UPDATE Employees SET Salary = Salary * 1.10 WHERE DepartmentID = 1;
INSERT INTO Audit (EmployeeID, Action, Date)
VALUES (123, 'Salary Increase', NOW());
COMMIT;

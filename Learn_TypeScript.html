Welcome to the TypeScript Tutorial

Learn TypeScript from basics to advanced topics

Introduction

TypeScript is a typed superset of JavaScript that compiles to plain JavaScript. It adds optional static types, classes, and interfaces to help you build robust, maintainable, and scalable applications.

TypeScript Basics

In this section, we will cover the basic concepts and syntax of TypeScript.

  • Type Annotations: Specify the type of variables, parameters, and return values.
  • Interfaces: Define custom types and ensure that objects conform to specific shapes.
  • Classes: Create reusable and extensible code using object-oriented principles.
<script type="text/typescript">
// Type Annotations
let message: string = "Hello, TypeScript";
let count: number = 10;
let isValid: boolean = true;

// Interface
interface Person {
  name: string;
  age: number;
}

let user: Person = {
  name: "John",
  age: 25
};

// Class
class Animal {
  name: string;

  constructor(name: string) {
    this.name = name;
  }

  speak(): void {
    console.log(`${this.name} makes a noise.`);
  }
}

let dog = new Animal("Dog");
dog.speak();
</script>

Type Annotations

TypeScript uses type annotations to specify the types of variables, parameters, and return values. This helps catch errors at compile time rather than at runtime.

  • Basic Types: string, number, boolean, etc.
  • Array Types: string[], number[], etc.
  • Tuple Types: [string, number], etc.
  • Union Types: string | number, etc.
<script type="text/typescript">
let username: string = "Alice";
let age: number = 30;
let isAdmin: boolean = true;

let scores: number[] = [90, 80, 85];
let userInfo: [string, number] = ["Bob", 45];

let id: string | number = "123";
id = 456; // valid
</script>

Interfaces

Interfaces define custom types and ensure that objects conform to specific shapes. They can also be used to describe the structure of classes.

  • Basic Interface: Define a simple interface with properties.
  • Optional Properties: Properties that may or may not be present.
  • Readonly Properties: Properties that cannot be changed after initial assignment.
<script type="text/typescript">
interface Car {
  brand: string;
  model: string;
  year: number;
  color?: string; // optional property
  readonly vin: string; // readonly property
}

let myCar: Car = {
  brand: "Toyota",
  model: "Corolla",
  year: 2020,
  vin: "12345ABC"
};
</script>

Classes

TypeScript supports object-oriented programming with classes. Classes can include properties and methods, and can be extended using inheritance.

  • Class Declaration: Define a class with a constructor and methods.
  • Inheritance: Create a class that extends another class.
  • Access Modifiers: public, private, and protected to control visibility.
<script type="text/typescript">
class Person {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  greet(): void {
    console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
  }
}

class Employee extends Person {
  position: string;

  constructor(name: string, age: number, position: string) {
    super(name, age);
    this.position = position;
  }

  work(): void {
    console.log(`${this.name} is working as a ${this.position}.`);
  }
}

let emp = new Employee("Alice", 30, "Engineer");
emp.greet();
emp.work();
</script>

Advanced TypeScript

In this section, we will cover more advanced topics in TypeScript, such as generics, namespaces, and modules.

  • Generics: Create reusable components that work with any data type.
  • Namespaces: Organize code into logical groups and prevent name collisions.
  • Modules: Split code into separate files and import/export functionality.
<script type="text/typescript">
// Generics
function identity(arg: T): T {
  return arg;
}

let output1 = identity("Hello");
let output2 = identity(100);

// Namespace
namespace MyNamespace {
  export class MyClass {
    constructor(public name: string) {}
  }
}

let myClassInstance = new MyNamespace.MyClass("Namespace Example");

// Module (in separate files)
// file: mathUtil.ts
export function add(a: number, b: number): number {
  return a + b;
}

// file: app.ts
import { add } from "./mathUtil";
console.log(add(2, 3)); // 5
</script>
